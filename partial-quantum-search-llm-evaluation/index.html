Partial Quantum Search LLM Evaluation

A controlled benchmark testing whether language models can apply partial quantum search formulas under the correct quantum query-cost model.

Core question

Can a model know the formula but fail the cost model?

The evaluation focuses on partial quantum search, where the mathematical formulas are short but the interpretation is fragile. A correct answer requires distinguishing global Grover iterations, local Grover iterations, parallel query depth, the standard Grover baseline, and the net saved-query count.

The main finding is that models often reproduced formulas or intermediate values correctly while still failing the operational query-complexity interpretation.

LLMs may reproduce the correct formulas while still failing the underlying cost model.

Gold answer

Base-case expected result

For \(N=10^6\), \(K=100\), and \(b=N/K=10,000\), the correct query accounting is:

Block size10,000
Global queries698.80
Local queries52.36
Total queries751.16
Standard Grover785.40
Queries saved34.24

The local-stage cost is not multiplied by \(K\) because the local searches are performed in parallel.

Findings

Observed failure patterns

Formula recall is not enough

Several responses used the correct formulas but still reached an incorrect query-saving conclusion.

Parallel cost was often treated as sequential

The most damaging base-case error was multiplying the local search cost by \(K=100\).

Small correction terms were dropped

Scale variants exposed models that rounded away correction terms decisive for the saved-query count.

Some models changed the baseline

When a result looked inconsistent, some models invented unsupported comparison baselines.

Critical labels used in review

Label
Meaning
C_FALSE_SAVING_CLAIM
Wrong final conclusion about whether queries are saved.
C_PARALLEL_AS_SEQUENTIAL
Treats parallel local search as sequential work by multiplying by \(K\).

Rounds away a correction term that determines the final saved-query count.

Compares against the wrong standard Grover-search baseline.
