Method

How the work is structured

The goal is not to collect isolated wrong answers. The goal is to identify repeatable failure mechanisms.

1

Define the failure target

Choose the exact behavior to test: baseline drift, correction-term loss, parallel-cost confusion, translation drift, or hallucination.

2

Build controlled variants

Keep the same structure while changing one assumption at a time. This makes the resulting failure interpretable.

3

Verify the gold answer

Expected answers are checked before model responses are scored. Numeric values and required concepts are stored separately.

4

Label model behavior

Conceptual, numeric, response-quality, and extraction errors are separated so the analysis does not overclaim.

5

Package evidence

Outputs are converted into reports, evidence logs, README files, and dataset-ready formats.
